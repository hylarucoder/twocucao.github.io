Tmux 与 Tmuxnator

发表于 2015-08-09 | 分类于 PC/MAC 使用心得技巧 |

这篇文章讲的究竟是什么？

如何优雅的开启 Django 开发。

笔者现在的技术栈为 Django+VueJS

在开发 Django 的时候，我们通常情况下是开启两个窗口

然后切到 PyCharm 或者 Spacemacs 里面编辑代码。

随着程序规模的增大和业务的复杂，往往环境的配置需要不同的配置文件和环境变量，而蛋疼的是，比如开启运行服务器，开启 Node 开发环境，开启 Celery+RabbitMQ, 开启各种各样的服务或者进程。

计算机一切的问题，都可以用手工暴力解决。

比如，每一次都手动开个七八个窗口，但这样做的缺陷是很明显的。

手工暴力解决实在是很吃力，而且容易出错。

为何不用工具来解决呢？

tmux + tmuxinator

什么是 Tmux?

其实就是在一个终端里开启多个终端的工具。

什么是 Tmuxinator?

就是 Ruby 开发的一个第三方包，用于方便快捷的建立多个终端的工具。

于是，你就可以在 Tmuxinator 里面编写好 YAML 文件，然后 start 一下，就出结果了。

这样开启多个 window, 多个 panel, 就很容易查看出程序编写过程中的问题。
